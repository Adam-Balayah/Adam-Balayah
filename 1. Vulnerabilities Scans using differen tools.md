#  Vulnerabilities Scans using different  tools
<br>

##  1. Conducting a Software Vulnerability Scan Using AlienVault


<p align="center">
   Navigate to the AlienVault website and enter the relevant credentials: <br>
  <img src="https://user-images.githubusercontent.com/121813491/210877292-5d4f18a0-6237-4aff-b17a-155d8ffa0dfd.png" alt="AlienVault login page">
</p>
 

<p align="center">
    In the AlienVault website dashboard, go to the Vulnerabilities section by hovering over the Environment menu and selecting the Vulnerabilities option.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210878417-71394fcc-05ec-4b4c-95f5-14cc87d40425.png" alt="AlienVault website dashboard">
</p>


<p align="center">
   In the Vulnerabilities window, go to the Scan Jobs section and select the New Scan Job option. <br>
  <img src="https://user-images.githubusercontent.com/121813491/210879473-3357a531-72ce-4193-bfc7-79b4b6f4c14f.png" alt="Vulnerabilities window">
</p>

<p align="center">
   In the Create Scan Job pop-up window, give the scan job a name and enter the IP address of the asset you want to scan in the search field. <br>
  <img src="https://user-images.githubusercontent.com/121813491/210879538-972f367f-f6ca-4ead-aaa9-e427494f18f8.png" alt="Scan Job pop-up window">
</p>

<p align="center">
   Save the scan job and select it to highlight it in blue. <br>
  <img src="https://user-images.githubusercontent.com/121813491/210879652-306db2f0-2419-4197-a0d0-3df8ab9bae0b.png" alt="Saved scan">
</p>

<p align="center">
   When the scan is complete, you can view the report by clicking on the PDF icon in the Action column.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210879727-e6cfed25-b8cd-4831-9401-151b48e05f4a.png" alt="Scan PDF">
</p>


<p align="left">
   Note: The report will open in a new tab and display a list of vulnerabilities detected on the Damn Vulnerable Web Application (DVWA), along with suggestions for how to fix them. This report can be used by a cyber security analyst to address the vulnerabilities.<br>
</p>

## 2. Conducting a Software Vulnerability Scan using Nessus


<p align="center">
   Log in to Nessus.<br>
  <img src="https://www.tenable.com/sites/default/files/images/articles/Nessus%20Essentials.png" alt="Nessus Login" width="60%" height="60%">
</p>


<p align="center">
   In the Nessus Web interface, go to the New Scan section and select the New Scan option.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882112-18050ac1-93b7-4c45-9997-074785d08d84.png" alt="Nessus Web interface">
</p>


<p align="center">
   In the Scanner window, choose the Web Application Tests option.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882492-93732a80-8e53-4bc9-b724-6f91e583dccd.png" alt="Nessus Scanner window">
</p>


<p align="center">
   In the New Scan/Web Applications window, give the scan a name, specify the target, and click Save.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882536-f037e2aa-dabb-4bbc-b336-98e46578ef30.png" alt="New Scan/Web Applications window">
</p>

<p align="center">
   In the My Scan window, select the newly created scan and go to the Launch option.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882643-cb3cb46e-c7de-4995-ae10-17be4dec5ed7.png" alt="Scan window">
</p>


<p align="center">
   Once the scan is complete, go to the Report menu and select the HTML option.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882703-9a1d34ab-39ad-49f2-b3d1-cc1200ae47af.png" alt="Report menu">
</p>


<p align="center">
   When the scan is complete, you can view the report by clicking on the PDF icon in the Action column.<br>
  <img src="https://user-images.githubusercontent.com/121813491/210882797-a554daf0-f943-4840-a059-49fbee1acbd4.png" alt="Scan column">
</p>

<p align="left">
 Note: The report will show a list of vulnerabilities detected by Nessus, with most of them listed under the Info tab. It is expected to see few or no errors in the report. If there are vulnerabilities that can be fixed, you can escalate them to the appropriate channel within the organization and review them in more detail. In the newly opened Microsoft Edge tab.<br>
</p>
